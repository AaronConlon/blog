实现一个Promise-A+规范的Promise类

JavaScript
最后更新: 2021/5/12

我之所以做这件事是因为我想更深入理解 Promise 的设计理念和设计原理，我看了一些文章和介绍，搜索了一些知识，也学习了其他人的 Promise实现源码。

这真的很令我振奋，通过实现一个符合Promise/A+规范的Promise类，我对Promise的理解有了不错的进步，那就废话少说，开始吧。

思考🤔

在我们开始之前，假设我们处于一个密闭空间，触手可及的只有手头的键盘。

想想，用少量词汇组织将要写的代码的内容，例如：

  • 我的 Promise 实现命名为Yi
  • Yi初始值为undefined，初始状态为pending，状态可以转变为fulfilled或者rejected,状态改变后不可逆，状态改变的逻辑只执行一次。
  • Yi具备两个静态方法：resolve和reject，当状态为pending时才执行逻辑，这样执行一次之后就直接返回，执行这两个方法都会

很快，我们的实现可能如下：

1function Yi(executor) {
2  this.state = 'pending'
3  this.value = undefined
4  executor(this.resolve.bind(this), this.reject.bind(this))
5}
6 
7Yi.prototype.resolve = function(value) {
8  if(this.state !== 'pending') return
9  this.state = 'fulfilled'
10  this.value = value
11}
12 
13Yi.prototype.reject = function(reason) {
14  if(this.state !== 'pending') return
15  this.state = 'rejected'
16  this.value = reason
17}

很棒，接着考虑其他功能，Yi应该实现then方法！

then方法是Promise/A+规范的核心部分。

那么，then方法必须提供的功能又有哪些才行呢？呃呃😯，想想可能会是：

  • then接收两个方法：onFulfilled和onRejected
  • 返回一个新的Yi（Promise）实例，这样可以进行链式调用。
  • 同一个Yi(Promise)实例的then方法可以执行很多次，每次都应该把新的Yi（Promise）实例保存到一个数组中，一个实例对象可能在其不同状态下注册了许多状态变更后的异步，我们将之交给then或者catch方法新生成的Yi(Promise)实例去执行。
  • Yi应该用一个数组保存then注册的状态变更回调任务，在合适的时候将当前Yi(Promise)实例的下一个状态的回调通过异步Api调用。

构造器接收的函数应该是一个executor函数参数，这个函数被同步执行。

那么，之前的代码应该改一改：

1function Yi(executor) {
2  this.state = 'pending'
3  this.value = undefined
4  this.consumers = []
5  executor(this.resolve.bind(this), this.reject.bind(this))
6}
7 
8Yi.prototype.resolve = function(value) {
9  if(this.state !== 'pending') return
10  this.state = 'fulfilled'
11  this.value = value
12  // 状态变更则广播给自己的 consumers
13  this.broadcast()
14}
15 
16Yi.prototype.reject = function(reason) {
17  if(this.state !== 'pending') return
18  this.state = 'rejected'
19  this.value = reason
20  // 状态变更则广播给自己的 consumers
21  this.broadcast()
22}

构造器内部用一个consumers数组保存then和catch实例方法中实例化的Yi(promise)对象。

在then方法中需要实例化一个新的Yi(Promise)对象，并且为之定义settled状态后的属性onFulfilled和onRejected，执行异步任务的逻辑，用一个broadcase函数封装起来：

1Yi.prototype.then = function(onFulfilled, onRejected) {
2  const promise = new Yi(function() {})
3  // 现在，这个返回且被保存到当前 Yi 实例的 consumers 数组中的新实例具有了两个状态稳定后的属性
4  promise.onFulfilled = typeof onFulfilled === 'function' ? onFulfilled : undefined
5  promise.onRejected = typeof onRejected === 'function' ? onRejected : undefined
6  this.consumers.push(promise)
7  // 发出广播，当旧的 Yi 实例为 settled 状态下的时候可以直接调用相关属性任务
8  this.broadcast()
9  return promise
10}
11 
12Yi.prototype.broadcast = function() {
13  const promise = this;
14  // called after promise is resolved
15  if(this.state === 'pending') return
16 
17  const callbackType = this.state === 'fulfilled' ? 'onFulfilled' : 'onRejected'
18  const toSettled = this.state === 'fulfilled' ? 'resolve' : 'reject'
19 
20  setTimeout(() => {
21    // 当状态改变后将此前 then 保存的 consumers 内的 promise 全部广播一遍状态变更的消息，并且清空此数组
22    promise.consumers.splice(0).forEach(consumer => {
23      try {
24        const callback = consumer[callbackType]
25        // 如果 promise 具有 onFulfilled 或者 onRejected 方法，在 then 方法执行的时候添加的
26        // 在 then 的时候判定是否为函数，不是函数则 callback 为 undefined
27        if(callback) {
28          consumer.resolve(callback(promise.value))
29        } else {
30          // callback 不是函数，那就默认按状态改变值
31          // 状态转为 settled
32          consumer[toSettled](promise.value)
33        }
34      } catch (e) {
35        // 异常捕获为 Rejected 状态下的 reason
36        consumer.reject(e)
37      }
38    })
39  }, 0);
40}

对于类似笔者这种功力不足的开发者来说，为了更好地实现Yi，理解Promise的原理，我想可以降低整体的复杂度，先用setTimeout模拟微任务，ok，最终版本一定会按平台通过nextTick或者MutationObserver来替换setTimeout。

话说回来，then方法中我们实例化一个不会在executor中同步让自己进入settled状态的Yi(promise)实例，并且根据当前的实例状态为这个新的实例添加了两个属性方法onFulfilled和onRejected，然后将之放入当前实例的consumers数组中暂存。

接着，我们调用了一个核心函数：broadcast。

瞧瞧其实现，当当前实例在pending状态下调用此方法，我们直接返回。

只有状态稳定了，我们再根据其状态fulfilled或者rejected来获取我们将要调用的异步任务函数，也就是在then方法的executor函数中为新Yi(Promise)实例创建的属性方法。

我们为初始化的Yi(Promise)实例通过then传入的状态变更处理函数onFulfilled和onRejected赋值给了内部新实例化，且在then中返回的Yi(Promise)实例。

在resolve、reject这两个可以变更实例状态的方法中调用了broadcast方法。

因为而状态变更会让所有的元素都执行then中添加的异步任务，并且我们的实例可以在不同时机多次调用then方法新增consumers的元素，因此需要在适当的时机清空当前实例的consumers数组。
