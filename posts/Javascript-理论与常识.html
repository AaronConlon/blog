2020年12月27日22:46:02,ES6 看起来已经不是新知识点了,本文从新版 ECMAScript 特性开始说起.

ES7(2018) 新特性

Array.prototype.includes能快速查找数组中是否包含某个元素,包括NaN,之前的indexOf方法也能检查数组中的元素,但是不支持NaN的检查.我们知道NaN !== NaN.

另外,JavaScript 支持了指数函数的中缀表示法

1let n = 2**3 
2=> 2*2*2 
3=> 8
4let a = 2
5a **= 2 // a = a * a
6let b = 3
7b **= 3 // b = b * b * b

ES8(2019) 新特性

Object.values和Object.entries.

返回接收参数(对象或者数组)的可枚举属性的值的数组,后者是键值对数组.

1let obj = {x: 'xx', y: 1}
2Object.values(obj) 
3=> ['xx', 1]
4Object.entries(obj)
5=> [['x', 'xx'], ['y', 1]]

也算是对之前只有Object.keys的补全吧.

在 chrome 下,Object 的这三个方法都不能取得原型链上的值

Object还增加了getOnwPropertyDescriptors方法.接收一个对象作为参数,返回此对象所有自身描述符,如果没有任何自身属性,返回空对象.IE不支持这个特性.

1const obj = { 
2  get es7() { return 7; },
3  get es8() { return 8; }
4};
5Object.getOwnPropertyDescriptors(obj);
6// {
7//   es7: {
8//     configurable: true,
9//     enumerable: true,
10//     get: function es7(){}, //the getter function
11//     set: undefined
12//   },
13//   es8: {
14//     configurable: true,
15//     enumerable: true,
16//     get: function es8(){}, //the getter function
17//     set: undefined
18//   }
19// }

字符串方面,增加了两个有趣的方法:

  • String.padStart(targetLength,[padString]) *targetLength：*当前字符串需要填充到的目标长度。如果这个数值小于当前字符串的长度，则返回当前字符串本身。

    padString：(可选)填充字符串。如果字符串太长，使填充后的字符串长度超过了目标长度，则只保留最左侧的部分，其他部分会被截断，此参数的缺省值为空格。

  • String.padEnd(targetLength,padString]) 参数释义同上。

我觉得在做cli输出数据,或者在table上能用到.

现在,对象和参数,数组都支持结尾逗号了.下面的代码都不会报错.

1//定义参数时
2function foo(
3    param1,
4    param2,//结尾逗号
5) {}
6//传参时
7foo(
8    'coffe',
9    '1891',//结尾逗号
10);
11//对象中
12let obj = {
13    "a": 'coffe',
14    "b": '1891',//结尾逗号
15};
16//数组中
17let arr = [
18    'coffe',
19    '1891',//结尾逗号
20];

如此一来有以下两个好处:

  • 重新排列项目更简单,如果最后一项改变位置,则不必删除逗号.
  • 帮助 git 等版本控制系统追踪实际发生的变化.
1[
2  'coffe'
3]
4// 变更
5[
6  'coffe',
7  '1995'
8]

现在采用结尾逗号,则在变动追踪的时候只会显示1995的变动,减少了无用信息的上报.

支持了异步函数.让我们能摆脱回调地狱(callback hell).

1async function asyncFunc() {
2    const result = await otherAsyncFunc();// otherAsyncFunc()返回一个Promise对象
3    console.log(result);
4}
5 
6// 等同于:
7function asyncFunc() {
8    return otherAsyncFunc()// otherAsyncFunc()返回一个Promise对象
9    .then(result => {
10        console.log(result);
11    });
12}

另外,ES8支持了共享内存和原子对象.

请移步看看这里,A cartoon intro to ArrayBuffers and SharedArrayBuffers - Mozilla Hacks - the Web developer blog.

SharedArrayBuffer对象用来表示通用固定长度的原始二进制数据缓冲区,类似ArrayBuffer对象.这里涉及到浏览器端的多线程work处理逻辑，共享数据和原子操作。

ES9如何？

首先，支持了异步迭代器。

1async function fn(arr) {
2    for (let i of arr) {
3        await someFn(i)
4    }
5}

上述代码依然无法让循环实现异步，而是同步执行，且在内部的异步函数得出结果之前全部调用完成。

1
2const promises = [
3    new Promise(resolve => resolve(1)),
4    new Promise(resolve => resolve(2)),
5    new Promise(resolve => resolve(3))
6]
7async function foo() {
8    for await (let i of promises) {        
9        console.log(i)
10    }
11}
12foo()
13=> 1,2,3

如果我们可能需要用到异步循环，注意引入ES9的支持。

ES9支持了 promise在最后时刻执行finally函数.

并且重新修订了字面量的转义,终于支持了String.raw.后续的字符串模版全部不会被转义.

1let s = `\u{54}` //会转义成unicode "T"
2console.log(s);//>> T
3 
4let str = String.raw`\u{54}`; //不会被转义
5console.log(str);//>> \u{54}

关于扩展运算符,ES6只支持数组的扩展.现在终于能操作对象了.

1const obj = {
2  a: 1,
3  b: 2,
4  c: 3
5};
6const { a, ...param } = obj; //这里...是rest
7console.log(a); //>> 1
8console.log(param); //>> {b: 2, c: 3}
9 
10function foo({ a, ...param }) {//这里...还是rest
11  console.log(a); //>> 1
12  console.log(param); //>> {b: 2, c: 3}
13}
14 
15const param = { b: 2, c: 3 };
16foo({ a: 1, ...param });  //此处...为spread

ES9还对正则表达式提供了非常给力的支持.例如之前.能匹配除了回车外的所有字符,现在添加flag标志s,就可以匹配回车了.

1/hello.world/s.test('hello\nworld') 
2=> true
3/a.b/s.text(`a
4b`)
5=> true

另外,对正则表达式命名捕获组也进行了支持:

1const reDate = /(\d{4})-(\d{2})-(\d{2})/,
2  match = reDate.exec("2018-08-06");
3console.log(match);//>> [2018-08-06, 2018, 08, 06]
4 
5//这样就可以直接用索引来获取年月日：
6let year = match[1]; //>> 2018
7let month = match[2]; //>> 08
8let day = match[3]; //>> 06
9// 一旦字符串顺序变化,则除了要修改正则表达式之外,还需要修改上面三行取值的代码.

ES9支持对匹配组进行命名,获取匹配结果的时候可以通过分组的名字获取.

1const reDate = /(?<year>\d{4})-(?<month>\d{2})-(?<day>\d{2})/,
2  match = reDate.exec("2018-08-06");
3console.log(match);
4//>> [2018-08-06, 08, 06, 2018, groups: {day: 06, month: 08, year: 2018}]
5 
6//此时用groups对象来获取年月日，无论正则表达式怎么变换，这下面三行不用改了，省事！
7let year = match.groups.year; //>> 2018
8let month = match.groups.month; //>> 08
9let day = match.groups.day; //>> 06

正则表达式的进阶小书里提及的断言,之前只支持正向现行断言,ES9支持了后行断言.
