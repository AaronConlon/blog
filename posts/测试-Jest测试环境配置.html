测试-Jest测试环境配置

Jest
Test
Configuration
最后更新: 2020/12/9

在一切开始之前,我已经全局安装了eslint和jest.那么直入正题,上代码:

1mkdir Demo && cd Demo
2# 初始化
3npm init -y
4jest --init
5# eslint 这里,选择了检查语法和修复,并且遵循 standard 的规范进行编码
6eslint --init

如果需要显示覆盖率，则修改 package.json 的 test script,修改为: "test": "jest --coverage"

接着便按照jest 的文档,试着写一个用例.在vscode中出现了诸如 describ 和test等未定义的eslint警告.

接着,需要安装一款插件:eslint-plugin-jest - npm,并且按官方文档进行配置,在 eslint 的配置文件这里增加插件项目:

1{
2  "plugins": ["jest"]
3}

以及添加规则,避免没必要的警告.

1{
2  "rules": {
3    "jest/no-disabled-tests": "warn",
4    "jest/no-focused-tests": "error",
5    "jest/no-identical-title": "error",
6    "jest/prefer-to-have-length": "warn",
7    "jest/valid-expect": "error"
8  }
9}

其实也可以通过配置环境变量项目告知 eslint 部分变量已经由jest提供.依然是编辑eslint的配置文件.

1{
2  "env": {
3    "jest/globals": true
4  }
5}

可以通过修改 jest 的配置文件,自动生成 web 的测试覆盖率文档和详解.

接着,还需要支持typescript.

1# 安装 ts 依赖和插件
2npm i ts-jest @types/jest typescript @types/node --save-dev
3# init ts config
4touch tsconfig.json
5# 如果全局安装了 tsc 则直接 tsc --init 即可创建初始化的 ts 配置文件

编辑配置文件 tsconfig.json:

1{
2  "compilerOptions": {
3    "target": "es5",
4    "module": "commonjs",
5    "lib": [
6      "es2015"
7    ],
8    "strict": true,
9    "declaration": true,
10    "outDir": "build",
11    "sourceMap": true
12  },
13  "include": [
14    "src/**/*",
15    "__test__/**/*"
16  ],
17  "exclude": [
18    "node_modules"
19  ]
20}

接下来这部分是我从图雀社区复制过来的.修改 jest.config.js 配置

添加如下配置项：

// An array of file extensions your modules use
2moduleFileExtensions: [
3    "js",
4    "json",
5    "jsx",
6    "ts",
7    "tsx",
8    "node"
9],
10// A preset that is used as a base for Jest's configuration
11preset: "ts-jest",
12

简单来说就是方便不用写引入的扩展名,以及为ts代码设置一个preset预处理,最后执行的是js代码.

自此,基础的jest测试环境配置完成,已经满足我小范围函数测试的需求了.
