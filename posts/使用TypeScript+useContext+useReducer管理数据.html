使用TypeScript+useContext+useReducer管理数据

TypeScript
状态管理
最后更新: 2021/12/24

前言

在不引入第三方库的情况下，React Context 可以使用内置的useContext和useReducer两个钩子来创建状态管理模块。

代码

首先，在项目根目录下创建contexts目录，内部存放所有的context，可以根据不同的用途创建不同的目录，内层创建三个文件：

  • context.ts 存放核心 context
  • reducer.ts 存放数据变更专用的 reducer
  • types.ts 存放 TypeScript 类型和接口定义

context

代码如下：

import React, { createContext, useContext } from "react";
2import { IState, IRootAction } from "./types";
3 
4// 定义初始状态值
5export const initState: IState = {
6  count: 0,
7  profile: {
8    name: "Aaron",
9    age: 100,
10  },
11};
12 
13// 定义 context，使用 createContext api
14export const RootContext = createContext<{
15  state: IState;
16  dispatch: React.Dispatch<IRootAction>;
17}>({
18  state: initState,
19  dispatch: () => null,
20});
21 
22// 导出 context 的 provider，用于放置 context
23export const RootContextProvider = RootContext.Provider;
24// 导出 hooks ，便于获取 state 和 dispatch 函数
25export const useRootContext = () => useContext(RootContext);

reducer

代码如下：

import { IState, IRootAction } from "./types";
2 
3// 定义 reducer 的参数类型
4export const rootReducer = (state: IState, action: IRootAction): IState => {
5  const { type, payload } = action;
6  // 根据类型进行处理，返回最终的 state
7  switch (type) {
8    case "add_count":
9      return {
10        ...state,
11        count: state.count + (payload as number),
12      };
13    case "sub_count":
14      return {
15        ...state,
16        count: state.count - (payload as number),
17      };
18    case "update_profile":
19      return {
20        ...state,
21        profile: payload as typeof state.profile,
22      };
23    default:
24      return state;
25  }
26};

types

共用的类型值：

export interface IState {
2  count: number;
3  profile?: {
4    name: string;
5    age: number;
6  };
7}
8 
9export interface IRootAction {
10  type: "add_count" | "sub_count" | "update_profile";
11  payload:
12    | number
13    | {
14        name: string;
15        age: number;
16      };
17}

用法和解析

此时，只要在顶层使用 provider 即可：

import { useReducer } from "react";
2import { Link } from "react-router-dom";
3import { BrowserRouter, Route, Routes } from "react-router-dom";
4 
5import "./App.css";
6 
7import Total from "routes/total";
8import Home from "routes/home";
9import { rootReducer } from "contexts/root/reducer";
10import { initState, RootContextProvider } from "contexts/root/context";
11 
12function App() {
13  const [state, dispatch] = useReducer(rootReducer, initState);
14 
15  return (
16    <RootContextProvider value={{ state, dispatch }}>
17      <BrowserRouter>
18        <div className="navs">
19          <Link to={"/"}>Home</Link>
20          <Link to={"/total"}>Total</Link>
21        </div>
22        <Routes>
23          <Route path={"/"} element={<Home />} />
24          <Route path={"/total"} element={<Total />} />
25        </Routes>
26      </BrowserRouter>
27    </RootContextProvider>
28  );
29}
30 
31export default App;

关键在于使用 useReducer传入reducer和默认数据，得到包含state和dispatch的数组，将二者传给Provider的value参数即可。

后续组件便可以如此：

import CountBtn from "components/CountBtn";
2import { useRootContext } from "../../contexts/root/context";
3import Profile from "./Profile";
4 
5export default function Total() {
6  const { state } = useRootContext();
7  const { profile } = state;
8 
9  return (
10    <div
11      style={{
12        margin: "2rem",
13      }}
14    >
15      <p>count is: {state.count}</p>
16      <CountBtn />
17      <p>
18        I am {profile?.name}, {profile?.age} years old.
19      </p>
20      <Profile profile={profile as any} />
21    </div>
22  );
23}

通过useRootContext钩子获取到state，同时也可以获取到dispatch，有需要就可以修改数据了。

重复渲染的问题

当组件内使用了useRootContext后，context内任意数据变化都会引起当前组件重新渲染，举个例子：

当我使用dispatch更新了count的值，某个使用了useRootContext的组件即使没有使用count，也会重新渲染。

这个问题，可以有两种类似的方法解决：

  • 方案 1：将组件返回的jsx封装成新的组件，将组件需要的数据通过props传入进去，并且此新组件通过React.memo包裹起来并导出，这时候传入的参数不变，则子组件不会重新渲染，这是利用了React.memo比较传入的props相等时不渲染的原理。
  • 方案 2：将使用了useRootContext的组件 return 的jsx部分替换成useMemo(() => 原先的jsx, [deps])，原先的jsx内依赖的数据全部放到依赖数组里，这是利用了useMemo缓存jsx的原理。

当然还有更好的方案，比如：

  • 拆分 Context ，构造一套更精细的数据更新控制机制，例如《如何避免 useContext 重渲染》中提及的思路
  • 使用第三方库，如Recoil、concent · power your react

总结

没啥好总结的，用Recoil或者concent吧。
