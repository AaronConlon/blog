使用react和svg创建圆环进度条

React
最后更新: 2021/9/10

前段时间刚好接触一个设计页面需要用到圆形进度环组件，当时使用的是谷歌的 material-ui 的组件，为了实现设计图的效果需要对库组件进行修改，于是搜索了关于圆形进度环的实现，学了一手使用 svg 的知识来创建这个“圆环”。

开始

1. 创建 React 基础环境

很久以前，我断断续续的学习了webpack相关知识，后来工作中接触不到相关知识便淡忘了所学的内容，再看webpack的时候忽然觉得模棱两可且所知甚少，如下再次配置一遍基础环境是为了让自己加深一些印象。

Create-react-app 很好，codesandbox.io也很方便，但是我还是想记录一下搭建简单的 React 开发环境。

首先，yarn init -y初始化根目录之后，即可安装以下依赖：

1# react 核心依赖
2yarn add react react-dom
3# babel 相关依赖
4yarn add -D babel-loader @babel/preset-env @babel/preset-react 
5# webpack 依赖
6yarn add webpack webpack-cli
7# 使用 webpack 初始化其配置文件，根据需求会自动安装一些依赖
8npx webpack init

接着，对.babelrc文件进行编辑使其支持react相关特性：

1{
2  "plugins": ["@babel/syntax-dynamic-import"],
3  "presets": [
4    [
5      "@babel/preset-env",
6      {
7        "modules": false
8      }
9    ],
10    ["@babel/preset-react"]
11  ]
12}

一切从简，接着安装@chakra-ui相关套件：

yarn add @chakra-ui/react @emotion/react@^11 @emotion/styled@^11 framer-motion@^4

熟悉了chakra-ui之后，可以很方便的直接写样式。

Material-UI 的设计很好，也许未来可以尝试在项目中使用。

ok，接下来正式开始介绍今天的主题：圆环进度条。

2. 圆环进度环组件设计 - svg

首先，我们来回顾一下最终效果：

如上所示，圆形中间有一个百分比字符串，圆形边缘是一个具有“进度”意义的紫色进度条，且其在整个环的占比为圆环中心的百分比字符串等值，并且其余百分比部分也有一个灰白色的样式。

svg 可以使用 fill 和 stroke 属性来上色，fill 设置对象内部的颜色， stroke 设置对象边框线条的颜色。

svg具有描边的概念，描边是以路径为中心线绘制的。我们的圆环进度就是利用圆形边线描边实现的。

描边不能与边框属性border混淆，利用svg描边我们可以创建一些优美的"边框"。

看看描边的三种方式：

我们的圆环就是使用 css 属性 troke-linecap: round;来实现的圆角，相较于默认的矩形边界，圆角看起来更柔和一些。

另外，圆环中心的文本使用text创建：

1<svg>
2    ...
3    <text className="svg-circle-text" x={}  y={}>
4        ...
5    </text>
6</svg>

整个组件的关键在于svg中使用两个圆形，一个作为百分比未达到的部分的背景，我们可以设置需要用到的背景色。另一个圆形则为了利用其描边，我们可以通过计算周长来设置描边数组：

1...
2const center = size / 2;
3const radiu = center - stokenW / 2;
4const circumference = 2 * Math.PI * radiu;
5...

描边使用了strokeWidth来设置描边的宽度，stroke-dasharray描边数组值，使用stroke-dashoffset来设置显示偏移错位。

按我的理解来看，stroke-dasharray的值作为总数，stroke-dashoffset则表示描边显示的范围，当偏移等于总数的时候，显示整个描边内容。

组件源代码如下：

1import { Box } from '@chakra-ui/layout';
2import React, { useEffect, useRef, useState } from 'react';
3import './style.scss';
4 
5export default function MyProgress(props) {
6  const {
7    size = 300,
8    progress = 0,
9    stokenW = 12,
10    circleOneStroke = '#eee',
11    circleTwoStroke = 'purple',
12  } = props;
13  const center = size / 2;
14  const radiu = center - stokenW / 2;
15  const circumference = 2 * Math.PI * radiu;
16  const [offset, setOffset] = useState(0);
17  const svgRef = useRef();
18  const oldProgress = useRef(progress);
19  const [time, setTime] = useState(1);
20 
21  useEffect(() => {
22    const newTime = Math.abs(oldProgress.current - progress);
23    oldProgress.current = progress;
24    setTime(12 * (100 - newTime));
25  }, [progress]);
26  
27  useEffect(() => {
28    const progressOffset = ((100 - progress) / 100) * circumference; // 百分比乘以圆周，等于当前百分比对应的显示部分值
29    setOffset(progressOffset);
30    svgRef.current.style = `transition: stroke-dashoffset ${time}ms ease-in-out;`;
31  }, [progress]);
32 
33  return (
34    <Box>
35      <svg className="svg" width={size} height={size}>
36        <circle
37          className="svg-circle-bg"
38          stroke={circleOneStroke}
39          cx={center}  // 圆心 x
40          cy={center}  // 圆心 y
41          r={radiu}    // 半径
42          strokeWidth={stokenW}  // 描边宽度，边线是描边的中心
43        />
44        <circle
45          className="svg-circle"
46          stroke={circleTwoStroke}
47          cx={center}
48          cy={center}
49          r={radiu}
50          strokeWidth={stokenW}
51          strokeDasharray={circumference}
52          strokeDashoffset={offset}
53          ref={svgRef}
54        />
55        <text
56          className="svg-circle-text"
57          x={center}
58          y={center}
59          dominantBaseline="middle"
60        >
61          {progress}%
62        </text>
63      </svg>
64    </Box>
65  );
66}
67

眼皮上下互博了，这个组件还有很多可以优化的地方。现在，父组件传递过来的progress百分比值的改动将会创建一个描边偏移的动画，在另外的场景下也许不需要这个动画，而是根据progress的值来动态调整偏移即可，因此说这个组件依然有很多值得优化的地方。
